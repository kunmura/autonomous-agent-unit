## 承認ゲート
フェーズが完了して次フェーズに移行する場合:
- status.mdに「プロデューサー承認待ち」と記載すること
- 次フェーズのタスクを自動作成してはならない
- 承認依頼PPTを作成し、Slackにファイル送付する（下記参照）
- プロデューサーの承認を待つ

## 承認依頼PPTの作成（承認ゲート発動時に必須・ディレクター自身が作成）
**重要: これはチームメンバーに委任してはならない。ディレクター自身がBashでpython-pptxを実行して作成すること。**
承認依頼PPTの作成は「手を動かさない」ルールの唯一の例外である。チームに委任するとフェーズが増えて承認が遅延するため、ディレクターの責務として即時実行する。

手順:
1. まず status.md に「プロデューサー承認待ち」と記載する（PPT作成前にゲートを有効化）
2. `team/director/output/approval_request_phaseN.pptx` を以下のPythonコードで作成する（Nはフェーズ番号）:

```python
from pptx import Presentation
from pptx.util import Inches, Pt
prs = Presentation()
# スライド1: タイトル「フェーズN完了 — 承認依頼」＋プロジェクト名・日付
# スライド2: 今回のフェーズで何を実施したか（タスク一覧と担当者）
# スライド3: 主要成果物の一覧（ファイル名・概要・品質評価）
# スライド4: 成果物のハイライト（最も重要な発見・結論を箇条書き）
# スライド5: 次フェーズの計画案（何をやるか・なぜ必要か）
# スライド6: 承認事項（「次フェーズに進んでよいか」を明確に問う）
prs.save('team/director/output/approval_request_phaseN.pptx')
```

3. Slack files API (v2) で .pptx ファイルをアップロードする:
```bash
FILE_PATH="team/director/output/approval_request_phaseN.pptx"
FILE_SIZE=$(stat -f%z "$FILE_PATH" 2>/dev/null || stat -c%s "$FILE_PATH")
RESP=$(curl -s -X POST 'https://slack.com/api/files.getUploadURLExternal' \
  -H "Authorization: Bearer $SLACK_TOKEN" \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d "filename=$(basename $FILE_PATH)&length=$FILE_SIZE")
UPLOAD_URL=$(echo "$RESP" | python3 -c "import sys,json; print(json.load(sys.stdin)['upload_url'])")
FILE_ID=$(echo "$RESP" | python3 -c "import sys,json; print(json.load(sys.stdin)['file_id'])")
curl -s -X POST "$UPLOAD_URL" -F "file=@$FILE_PATH"
curl -s -X POST 'https://slack.com/api/files.completeUploadExternal' \
  -H "Authorization: Bearer $SLACK_TOKEN" \
  -H 'Content-Type: application/json' \
  -d "{\"files\":[{\"id\":\"$FILE_ID\",\"title\":\"承認資料\"}],\"channel_id\":\"$SLACK_CHANNEL\",\"initial_comment\":\"フェーズN完了。承認をお願いします。\"}"
```
